## PR Description
Fix a panic in `handle_sio_event` where a naked byte slice `&data.to_string()[..data.to_string().len().min(500)]` panicked with `byte index 500 is not a char boundary` when a multi-byte UTF-8 char straddled byte 500. Real-world Sentry repro: WhatsApp ingest payload with Cyrillic `'н'` at bytes 499..501.

The fix binds `data.to_string()` once and removes raw payload content from logs entirely (PII/security policy). The debug log now only emits the event name, not payload content. A regression test is included.

**Note on PR description vs actual code**: The "Solution" section of the PR description says it uses `floor_char_boundary` to safely slice the payload for logging. However, the actual final code (after CodeRabbit feedback) takes a stronger approach — it removes the payload content from debug logs entirely. The PR evolved across 3 commits: initial fix used `floor_char_boundary`, CodeRabbit flagged PII concerns, author removed payload from logs completely. The "Solution" section describes the initial commit, not the final state.
