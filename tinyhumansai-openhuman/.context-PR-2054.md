## PR Description
Adds `[Cross-chat context]` block to the agent prompt drawn from the same user's other JSONL chat threads, with provenance tags. Two complementary paths:

1. **Primary (live chats):** JSONL substring scan over the user's `workspace/memory/conversations/threads/*.jsonl`, scoring each message by the fraction of query terms it contains (terms >= 3 chars). Hits are filtered by `min_relevance_score`, capped at `CROSS_CHAT_LIMIT`, snippet-truncated, and rendered as `- [<provenance>] <snippet>` lines.

2. **Fallback (tests / headless):** when `workspace_dir` is not wired, fall back to `Memory::recall(query, opts.cross_session=true)`, which calls into existing `UnifiedMemory::episodic_cross_session_search` FTS5 helper.

Privacy/scope: JSONL directory is per-user, active thread excluded via `thread_context::current_thread_id()`. Provenance tags use hashed session IDs (never raw).

Manual smoke test documented: built OpenHuman.app (debug), started two threads with overlapping topics, third thread recovered context from the others.
