## AI Summary
**What it does**: Adds a cost visibility dashboard to the Settings surface, surfacing 7-day spend trends, per-model token/cost breakdown, and budget utilization with real-time polling. Wires cost telemetry into the agent turn loop via a process-global singleton so data accumulates automatically from every provider call.

**Breaking risk**: Medium. The `cost.enabled` default change from `false` to `true` is a silent opt-in for existing deployments. The `record_usage_unconditional` path changes the semantics of `cost.enabled = false` from "no tracking" to "no enforcement" — a behavior change that affects users who explicitly disabled cost tracking for privacy or performance reasons.

**Security risk**: Low. No secrets, no auth bypass, no PII in logs (model names and token counts are not sensitive). JSONL data is stored locally only.

**Bottom line**: Not safe to merge as-is — the behavioral semantics change on `cost.enabled = false` needs user-visible signaling, and the default flip needs a startup log notice so operators aren't caught off guard on upgrade.
