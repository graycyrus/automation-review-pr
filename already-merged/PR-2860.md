## AI Summary

**What it does**: Follow-up security hardening to the artifacts persistence layer from #2801. Rejects `"."` as an artifact ID (which previously resolved to the artifacts root directory), adds a `starts_with` path traversal guard in `ai_get_artifact` to block corrupt/adversarial `meta.path` values, flattens the `get_artifact` schema output to match the actual JSON response shape, and adds a test for the dot-rejection.

**Breaking risk**: Low — the schema fix makes declared output more accurate, not less. The `"."` rejection and path guard only affect previously-broken or adversarial inputs.

**Security risk**: Medium — the `Path::starts_with` guard in `ops.rs` is incomplete. `Path::starts_with` is a lexical component-prefix check and does NOT normalize `..` segments. A `meta.path` of `"../x"` produces `artifacts_root.join("../x")` whose components still start with `artifacts_root`'s components, so the guard passes and returns an out-of-root path. The proper fix is to reject paths that are absolute or contain any `ParentDir` component (as CodeRabbit suggested). This is a residual vulnerability.

**Bottom line**: Not fully safe — the path traversal guard has a bypass via `..` components. Needs a follow-up fix.
